# PAL95 兼容推进清单

## 1. 当前目标

`dos6.22.img` 已经可以启动，`pal95/` 也已经能挂成 `hda`。接下来《仙剑奇侠传》专项支持要解决的问题，不再是“能不能看到文件”，而是“DOS 游戏运行时依赖的环境是否足够像一台当年的 PC”。

## 2. 当前已具备

### 2.1 硬盘装载

- 已支持把 `download/pal95/` 根目录文件打成 FAT16 数据盘。
- 已支持以 `hda` 挂到 `v86`，并在 DOS 中通过 `C:` 访问。
- 已额外注入：
  - `RUNPAL.BAT`
  - `PALDIAG.BAT`
  - `PALREADME.TXT`

### 2.2 内存

- 页面中的内存参数已经会真实传给 `v86 memory_size`。
- 当前更接近“整机总内存”控制，而不是 DOS 层的 XMS/EMS 驱动控制。
- 如果游戏后续提示扩展内存不足，需要继续确认：
  - 启动镜像是否已加载 `HIMEM.SYS`
  - 是否需要 `EMM386.EXE`
  - `CONFIG.SYS` / `AUTOEXEC.BAT` 是否需要仙剑专项配置

### 2.3 VGA

- 已具备 VGA BIOS。
- 已能监听并记录 `screen-set-size` 事件。
- 这足以帮助我们确认游戏是否切到了图形模式，尤其是 320x200 / 256 色路径。

### 2.4 声卡

- 已支持页面级的 SB16 / AdLib 开关。
- 已在 `RUNPAL.BAT` 中注入 `BLASTER=A220 I7 D1 H5 T6`。
- 这一步能先验证游戏是否会尝试初始化数字音频与 FM 音源。

### 2.5 时序

- 当前先依赖 `v86` 默认 CPU/PIT 行为。
- 已支持按 CPU 档位映射 CPUID 等级，并保留日志观察点。
- 如果后续出现：
  - 过场速度异常
  - 输入延迟明显
  - 音乐节拍不稳
  - 特定场景卡死

  则要继续做专项时序兼容分析。

## 3. 当前仍未完全解决

### 3.1 XMS / EMS 驱动状态

《仙剑奇侠传》这类 DOS 游戏通常不会只看“总内存大小”，还可能依赖 DOS 启动环境里是否真的提供 XMS/EMS。当前项目还没有直接改写 `dos6.22.img` 内的系统配置，因此这部分仍属于下一阶段重点。

### 3.2 游戏内声音实测

现在只能确认：

- 仿真器侧已经暴露声音能力入口。
- 启动批处理会给游戏提供常见的 `BLASTER` 环境变量。

但是否完全兼容，还需要进入游戏后验证：

- SB16 数字音频
- AdLib / FM 背景音乐
- IRQ / DMA 配置匹配情况

### 3.3 运行期时序调优

如果《仙剑》能进标题甚至能进场景，但动画、移动、音乐或战斗出现快慢异常，就说明我们已经越过“启动兼容”阶段，进入“时序兼容”阶段。这时需要针对具体症状继续做采样和调优。

## 4. 推荐验证顺序

1. 启动 `dos6.22.img`，挂载 `pal95` 数据盘。
2. 在 DOS 中先执行 `PALDIAG`。
3. 观察：
   - `MEM` 输出
   - `BLASTER` 是否存在
   - `PAL.EXE` 是否可见
4. 再执行 `RUNPAL`。
5. 结合前端日志判断：
   - 是否发生 VGA 图形模式切换
   - 是否在切图后卡死
   - 是否出现声音初始化后异常

## 5. 下一步实现优先级

1. 若 `RUNPAL` 已能进入游戏：优先验证声卡和时序。
2. 若 `RUNPAL` 启动即报内存问题：优先改造 DOS 启动镜像中的 XMS/EMS 配置。
3. 若进入图形模式后黑屏：优先分析 VGA 模式切换后的兼容行为。
4. 若有画面但无声：优先补齐声卡初始化与游戏内默认设置引导。
