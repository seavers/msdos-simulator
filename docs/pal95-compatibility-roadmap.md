# PAL95 兼容推进清单

## 1. 当前基线

现在 PAL95 的运行方案已经改成：

1. 系统盘从 `storage/images/` 读取原始 DOS 启动镜像。
2. 启动前按页面参数生成 `startupDisk`，只改 `CONFIG.SYS`、`AUTOEXEC.BAT` 和启动辅助文件。
3. 扩展硬盘从 `storage/extendDisk/` 根目录直接读取原始镜像并挂载。
4. 不再生成 `generated/pal95-hdd.img`，也不再改写 PAL95 原始镜像内容。

## 2. 当前已经明确的结论

### 2.1 DOSIDLE

- 开关对 PAL95 能否启动影响不大。

### 2.2 CD 驱动 / MSCDEX

- 去掉 CD 驱动与 `MSCDEX` 后，PAL95 已经可以进入游戏画面。
- 当前页面已经把底层 `CD1.SYS` 和 `MSCDEX` 拆成两个独立开关，可以继续分别定位问题来源。

### 2.3 启动脚本职责

- `RUNSAFE.BAT`、`RUNPAL.BAT`、`PALDIAG.BAT` 现在都放在 `A:`。
- 它们只负责切盘、设环境变量、做诊断，不再修改 `C:` 里的 `SETUP.DAT`。

## 3. 当前仍需继续验证的点

### 3.1 PAL95 原始镜像约束

因为我们不再改写原始镜像，所以需要确认：

1. 原始镜像是否能稳定挂成 `C:`。
2. `PAL.EXE` 是否位于脚本可直接进入的位置。
3. `SETUP.DAT` 是否已经是你期望的配置。

### 3.2 图形模式与显示

- 当前已经能看到游戏画面，但仍需继续确认是否存在只显示一半、比例不正确或图形模式切换异常。
- 这部分要继续结合 `screen-set-size` 日志和实际画面观察。

### 3.3 声卡

- 现在的声音模式只通过 `BLASTER`、`SOUND`、`MIDI` 环境变量切换。
- 若 PAL95 对 `SETUP.DAT` 有强依赖，则需要你先在线下处理原始镜像。

## 4. 推荐排查顺序

1. 用最小系统盘脚本启动：
   - 常规内存优化：开
   - DOSIDLE：关
   - CD 驱动：关
   - MSCDEX：关
2. 先验证 `RUNSAFE` 能否稳定进画面。
3. 再分别打开：
   - `CD1.SYS`
   - `MSCDEX`
4. 最后再验证 `RUNPAL` 的声音路径。

## 5. 如果线下需要调整原始镜像

建议优先调整：

1. `C:\PAL.EXE` 是否可以直接运行。
2. `SETUP.DAT` 是否已经符合你要测试的声卡配置。
3. 是否需要把额外启动 BAT 或诊断脚本预先写入原始镜像。

如果原始镜像不能满足这些前提，当前仓库会继续保持“启动盘做脚本，原始扩展盘不动”的原则，不会自动替你重打游戏盘。
