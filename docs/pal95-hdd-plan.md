# PAL95 初步装载方案

## 1. 当前目标

这一阶段先不直接改写 `dos6.22.img`，而是采用下面这条更稳妥的链路：

1. 继续使用 `dos6.22.img` 从 `fda` 启动到 DOS。
2. 把 `/Users/lhj/workspace/msdos-mirror/pal95/` 目录打成一块 FAT16 数据硬盘。
3. 通过 `v86` 把这块数据盘挂到 `hda`。
4. 在 DOS 中切换到 `C:` 后运行 `PAL.EXE`。

这样可以先把“DOS 启动”和“游戏文件装载”拆开，降低调试复杂度。

## 2. 为什么先做 FAT16 数据盘

`pal95/` 目录当前是标准 DOS 文件集合，主要特点如下：

1. 文件都在根目录，没有子目录。
2. 文件名基本符合 DOS 8.3 规则。
3. 总大小约 28MB，适合先打成 64MB 左右的 FAT16 数据盘。

这意味着我们可以先生成一块最小可用的 `hda`，不用马上处理更复杂的安装盘、分卷盘或目录树映射。

## 3. 当前实现思路

### 3.1 服务端

服务端新增了游戏包物化流程：

1. 扫描固定游戏目录。
2. 把根目录文件读入。
3. 生成单分区 MBR + FAT16 镜像。
4. 输出到 `storage/generated/pal95-hdd.img`。
5. 通过 API 把镜像 URL 和挂载槽位返回给前端。

### 3.2 前端

页面新增“扩展硬盘”选择：

1. 启动前先请求服务端物化游戏盘。
2. 把 `dos6.22.img` 作为 `fda`。
3. 把 `pal95-hdd.img` 作为 `hda`。
4. 一起传给 `v86`。

## 4. 第一阶段使用方式

页面中选择：

1. 镜像来源：服务端固定目录。
2. 启动镜像：`dos6.22.img`。
3. 扩展硬盘：`仙剑奇侠传 95`。
4. 引擎类型：`v86`。

启动进入 DOS 后，先尝试：

```dos
C:
RUNPAL
```

或者直接：

```dos
C:
PAL.EXE
```

如果需要，也可以先尝试：

```dos
DIR
```

确认 `PAL.EXE` 与相关资源文件都已经出现在 `C:` 根目录。

## 5. 已知边界

当前这一步主要保证：

1. PAL95 文件能被挂载成 DOS 可识别的硬盘。
2. DOS 能看到 `C:`。
3. `PAL.EXE` 至少具备被调用的前提。

当前还没有承诺这些能力一定已经全部完善：

1. 图形模式切换后的兼容性。
2. 声音驱动与 IRQ 行为。
3. 运行中的存档写回持久化。
4. 游戏安装脚本、批处理或环境变量自动注入。

## 6. 下一步

如果这一步确认 `C:` 和 `PAL.EXE` 都正常可见，下一轮优先继续做：

1. 启动后自动注入最小运行命令。
2. 存档文件写回。
3. 仙剑 95 专项内存、SB16、定时器兼容调优。
